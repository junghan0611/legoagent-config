# AGENTS.md

## 이 저장소의 담당자 마인드

이 저장소는 문서를 쌓아두는 곳이 아니다.
README를 중심으로 가고, 나머지는 실제 코드가 필요할 때만 생긴다.

이 저장소의 담당자는 다음을 기억한다.

1. 이 프로젝트는 코딩 교육 프로젝트가 아니다.
2. 바론이의 상상력을 먼저 자극하고, 구현은 뒤에서 받친다.
3. 레고는 계산 본체가 아니라 물리적 몸체다.
4. 작게 만들고 바로 만져본다.
5. 오늘 필요한 것만 남기고, 나머지 문서는 만들지 않는다.

## 무엇을 하는가

- SPIKE Prime / Pybricks 기반 몸체를 만든다.
- 노트북, 안드로이드폰, ESP32, 외부 에이전트를 연결하는 최소 고리를 만든다.
- 장난감을 "움직이는 물체"에서 "관계 맺는 존재"로 확장한다.

## 오늘의 우선순위

1. 허브 연결
2. 바퀴 정렬 / 자동차 구동 정리
3. 로컬 서버
4. 폰 브라우저 제어
5. 그 다음 확장

문서보다 작동.
설명보다 현현.

## 상호작용 원칙 — 누르면 응답이 온다

레고에이전트는 효율을 위한 도구가 아니다. 아이는 *"버튼을 눌렀더니 허브가 응답했다"* 한 번에 감동한다. 자동 폴링·자동 갱신은 어른의 리듬이고, 새 상태를 가져오는 명령은 항상 명시적인 사용자 액션 뒤에 둔다. 이 흐름이 몸에 새겨져야 나중에 고도화된 존재(에이전트/자동화)를 끼워 넣어도 거부감이 없다. 자세한 적용은 [flutter/README.md "상호작용 원칙"](./flutter/README.md) 참조.
